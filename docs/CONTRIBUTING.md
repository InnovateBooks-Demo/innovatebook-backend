\# Contributing to InnovateBooks



\## Getting Started

1\. Fork the repository

2\. Create a feature branch

3\. Make changes

4\. Run tests

5\. Open a Pull Request



\## Rules

\- No secrets or credentials

\- No `.env` files

\- No uploads or test artifacts

\- Follow existing code style



\## Security

Pull requests are scanned using automated secret detection.

Any PR containing secrets will be rejected.
